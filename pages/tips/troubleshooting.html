General tips to keep in mind

There are many different things that could cause a problem with your AbanteCart installation.

Mod_security

hosting related
Mod_security is an apache module that helps to protect your website from various attacks. mod_security has certain rules for the words "pattern" used in requested URL's. If any pattern match with the requested URL under your site mod_security false positive and deny the access to it. Many people are experiencing problems because of this. Problems include "403", "500" or access denied errors (forbidden access), login problems, and similar. Ask your hosting support to disable individual mod_security rules or try to disable it
Try adding this in a .htaccess file in the /admin/ folder 

	
SecFilterInheritance Off

503 error

server related
Most of the time, a 503 error occurs because the server is too busy or because there's maintenance being performed on it.

Too many redirects occurred, log-in and log-out problems

PHP accelerators
PHP accelerators can substantially increase Opencart websites speed but apparently caching plugins sometimes causes problems with shopping carts.

GD

content delivery network (CDN)
GD php library is installed

ZIP

extension
ZipArchive extension is enabled

AbanteCart will not work properly (or at all) without these technologies.
